# Learning Loop

How to use this app as an actual learning tool, not just read-through material.

## The loop

1. **Read the lesson** — skim once to get the concept, then read again slowly.
2. **Open a real terminal** — run every example yourself. Don't skip this.
3. **Work the exercise** — each step shows a scenario. Try the command in your terminal before looking at the expected answer.
4. **Take notes** — use the notes area for anything that surprised you, confused you, or that you want to remember. Even one sentence is enough.
5. **Mark complete** — only after you've run the commands yourself.
6. **Come back** — revisit lessons when you forget something. That's normal and expected.

## What "done" means

A lesson is done when you can run the command from memory in a fresh terminal and explain what it does out loud. Not when you've read it.

## Going further

After each lesson, try variations on your own system:
- After `ls`: try `ls /etc` or `ls /var/log`
- After `cd`: navigate to a directory 3+ levels deep without using tab-complete
- After `pwd`: open two terminals, navigate to different locations, check each with `pwd`
